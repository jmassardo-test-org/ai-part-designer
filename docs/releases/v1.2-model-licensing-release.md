# Epic 13: Model Licensing & IP Protection — Release Summary

## v1.2.0 Release — February 26, 2026

### Overview

Epic 13 adds Creative Commons licensing, remix enforcement, and IP violation reporting to the marketplace. This is a **backend-only** release (Phase 1) — no frontend changes are included.

---

## What's Included

### License Catalog

Nine standard license types available via `GET /api/v2/licenses/types`:

| License | Commercial | Remix | Share-Alike |
|---------|-----------|-------|-------------|
| CC0 (Public Domain) | ✅ | ✅ | — |
| CC BY | ✅ | ✅ | — |
| CC BY-SA | ✅ | ✅ | ✅ |
| CC BY-NC | ❌ | ✅ | — |
| CC BY-NC-SA | ❌ | ✅ | ✅ |
| CC BY-ND | ✅ | ❌ | — |
| CC BY-NC-ND | ❌ | ❌ | — |
| All Rights Reserved | ❌ | ❌ | — |
| Custom | Configurable | Configurable | — |

### New API Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v2/licenses/types` | Public | License catalog |
| GET | `/api/v2/licenses/my/published` | User | User's published designs with licenses |
| GET | `/api/v2/licenses/my/remixed` | User | User's remixed designs with attribution |
| POST | `/api/v2/admin/designs/{id}/takedown` | Admin | Take down a design |
| GET | `/api/v2/admin/license-violations` | Admin | List violation reports |
| POST | `/api/v2/marketplace/designs/{id}/report-violation` | User | File a violation report (5/hr limit) |

### Marketplace Enhancements

- **Browse filters**: `license_type`, `allows_remix`, `allows_commercial` query params
- **Publish with license**: `license_type` + optional `custom_license_text`/`custom_allows_remix`
- **Remix enforcement**: Blocks remix for ND/All-Rights-Reserved licenses
- **Attribution**: Auto-injected on remix
- **Design detail**: Includes `license_info` object

### Rate Limiting

The violation report endpoint enforces 5 requests per hour per user.

## Database Migrations

| Migration | Description |
|-----------|-------------|
| `029_design_license_columns` | Adds `license_type` (String30), `custom_license_text` (Text), `custom_allows_remix` (Boolean) to `designs` |
| `030_content_report_evidence_url` | Adds `evidence_url` (Text) to `content_reports` |

Both migrations are **idempotent** — safe to run multiple times. They execute automatically via `alembic upgrade head` in the existing migration Job.

---

## Infrastructure Changes

- **No new environment variables, secrets, or ConfigMap entries**
- **No new external service dependencies**
- **Prometheus alerts added**:
  - `LicenseViolationRateHigh`: Fires if >20 violation reports filed in 1 hour
  - `LicenseAdminTakedownRateHigh`: Fires if >10 admin takedowns in 1 hour
- All new endpoints are automatically instrumented by existing Prometheus HTTP metrics
